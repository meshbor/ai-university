Артикли: a / an / the / —

Алгоритм принятия решения + частные случаи рабочей переписки. По материалам British Council: Articles и справочника по the.

Алгоритм (4 вопроса)

  1. Существительное в единственном числе и исчисляемое?
    Если да — артикль (или my/this/that…) обязателен. Голое bug, museum, report — всегда ошибка.
  2. Читатель понимает, о каком именно предмете речь? (уже упоминался / он единственный в контексте / есть уточнение «который…») → the
    the server (наш сервер, оба знают какой), the bug I mentioned
  3. Не понимает — предмет новый или «один из многих»? → a / an по первому звуку, не букве:
    a bug, an error, an hour (h немое), a user (звук «ю» = согласный j)
  4. Множественное число или неисчисляемое, и речь «вообще»? → без артикля
    Logs are useful. Caching improves performance.

Правило повествования: a → the

Первое упоминание — a/an, все последующие — the:

I found a bug in the API. The bug appears when a user sends an empty request.

Это правило закрывает большинство артиклей в любом письме, отчёте или issue.

Частные случаи для рабочих текстов

СлучайАртикльПример
Уникальные в контексте вещиthethe internet, the database, the team, the office
Превосходная степеньthethe fastest solution, the best way
Имена людей, городов, компаний—Rowan, London, Google (но: the UK, the USA)
Известные здания, музеи, организацииthethe British Museum, the EU
Профессииa/anI am a developer. She is an engineer.
Абстрактное/неисчисляемое «вообще»—performance, security, feedback, information

Самопроверка при письме

Закончив текст, найдите каждое существительное в единственном числе. Если перед ним нет артикля и нет my/this/that — это почти наверняка ошибка. Один такой проход ловит большинство пропущенных артиклей.
