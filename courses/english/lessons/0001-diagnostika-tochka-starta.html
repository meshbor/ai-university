Урок 0001 — Диагностика: точка старта

Где вы сейчас на самом деле — чтобы каждый следующий урок попадал точно в цель.

≈ 12 минут · чтение + грамматика + мини-письмо · справочник по уровням CEFR

Ваша миссия — читать техническую документацию без напряжения и уверенно писать рабочие тексты. Давний пробный IELTS ≈ 4 — это ориентировочно уровень B1 (официальная шкала IELTS ↔ CEFR), но навыки заржавели, и единственный честный способ узнать точку старта — проверить. Этим и займёмся.

Часть 1 · Чтение технического текста

Прочитайте текст один раз, не возвращаясь назад — как читали бы документацию по работе. Затем ответьте на вопросы по памяти. Это и есть проверка реального чтения, а не поиска ответов в тексте.

Caching is a technique that stores copies of data in a fast storage layer so that future requests can be served more quickly. When an application receives a request, it first checks the cache. If the data is there — a situation known as a cache hit — the application returns it immediately.

If the data is not in the cache — a cache miss — the application fetches it from the slower main database, returns it to the user, and usually saves a copy in the cache for next time.

Caches have limited space, so old or rarely used entries must be removed. This process is called eviction. A common strategy is LRU (Least Recently Used), which removes the entry that was used least recently.

1. What is a “cache hit”?
Верно. Hit — данные нашлись в кэше.
Не то. Перечитайте первый абзац: “If the data is there…”
2. What usually happens after a cache miss?
Верно. После промаха данные достают из БД и кладут копию в кэш.
Не то. Второй абзац: “…and usually saves a copy in the cache for next time.”
3. Which entry does the LRU strategy remove?
Верно. Least Recently Used — дольше всех не использовалась.
Не то. LRU = Least Recently Used — та, что дольше всех не использовалась.

Часть 2 · Грамматика и устойчивые сочетания

Шесть предложений из типичной рабочей переписки. Выберите вариант, который звучит правильно. Здесь спрятаны самые частые «ржавые» места: артикли, предлоги, времена.

1. The result on the input size.
Верно: he/she/it → depends; и это устойчивое сочетание depends on.
Нет. Подлежащее “the result” — третье лицо, ед. число.
2. I have worked with this codebase 2019.
Верно: have worked + since + точка во времени.
Нет. С Present Perfect и точкой отсчёта (2019) идёт “since”.
3. We the bug yesterday.
Верно: “yesterday” — законченное время, значит Past Simple.
Нет. Слово “yesterday” требует Past Simple, не Present Perfect.
4. The server returned unexpected error.
Верно: “unexpected” начинается с гласного звука → an.
Нет. Ошибка упоминается впервые и слово начинается с гласного звука.
5. This library consists three modules.
Верно: устойчивое сочетание — consists of.
Нет. По-русски «состоит из», но по-английски — consists of.
6. Let me know if you have questions.
Верно: в условных и вопросительных контекстах — any.
Нет. В “if”-предложениях обычно “any”. Это стандартная фраза переписки.

Часть 3 · Мини-письмо (внешняя проверка)

Последний шаг диагностики — письмо, ваш приоритетный навык. Статичная страница не оценит свободный текст, поэтому используем бесплатный инструмент Кембриджа, который ставит оценку по шкале CEFR.

  1. Откройте Write & Improve (регистрация не нужна — кнопка “Start writing”).
  2. Создайте свою задачу (“My own task”) и напишите 4–6 предложений по-английски на тему: “What I do at work and why I need English.” Без черновика на русском и без переводчика — пишите как получится, это диагностика, а не экзамен.
  3. Нажмите “Check” и посмотрите, какой уровень CEFR покажет система.

Когда закончите — напишите в чат свой результат: счёт из частей 1–2 (страница покажет его ниже кнопки) и уровень из Write & Improve. По этим двум цифрам я откалибрую все следующие уроки.
