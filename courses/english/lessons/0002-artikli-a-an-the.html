Урок 0002 — Артикли: a / an / the / —

Одна ошибка из вашего письма — и алгоритм, который закрывает её навсегда.

≈ 12 минут · после урока 0001 · шпаргалка: справочник по артиклям

В вашем диагностическом письме была фраза I was in interesting museum. Носитель спотыкается об неё мгновенно: пропал артикль. В русском артиклей нет, поэтому мозг их не «слышит» — это ошибка №1 у русскоговорящих, и хорошая новость в том, что она лечится одним простым алгоритмом.

Алгоритм: 4 вопроса

Перед каждым существительным задайте вопросы по порядку (источник: British Council):

  1. Единственное число и исчисляемое? Тогда артикль обязателен. Голое museum, bug, report — всегда ошибка.
  2. Читатель знает, о каком именно предмете речь? (упоминался раньше / единственный в контексте) → the: the server, the bug I mentioned
  3. Новый или «один из многих»? → a/an по первому звуку: a bug, an error, an hour (h немое), a user (звук «ю»)
  4. Множественное или неисчисляемое, и речь «вообще»? → без артикля: Logs are useful

И главное правило любого связного текста — a → the: первое упоминание получает a/an, все следующие — the:

I found a bug. The bug crashes the app.

Практика 1 · История одного бага

Связный текст — как настоящий отчёт коллеге. Следите за правилом a → the.

Yesterday I found bug in our API. Верно: первое упоминание — a. Нет: баг упоминается впервые → a. bug appears when Верно: теперь читатель знает, какой это баг — the. Нет: баг уже упоминался → The. user sends Верно: любой пользователь, «один из многих» — a. Звук «ю» = согласный. Нет: любой пользователь → a (user начинается со звука «ю» — согласного). empty request. Верно: empty начинается с гласного звука — an. Нет: запрос новый, empty начинается с гласного звука → an. I checked logs from yesterday and wrote Верно: конкретные логи (вчерашние, с нашего сервера) — the. Нет: это конкретные логи — вчерашние → the. short report. Верно: отчёт появился впервые — a. Нет: отчёт упоминается впервые → a.

Практика 2 · Отдельные предложения

Здесь вперемешку все четыре исхода — включая «без артикля». Не торопитесь, прогоняйте алгоритм.

1. Last weekend I went to interesting museum.
Верно. Та самая фраза из вашего письма — теперь она правильная.
Нет. Музей новый для читателя, interesting начинается с гласного → an.
2. It was British Museum.
Верно. Известные музеи и здания — с the: the British Museum.
Нет. Известные музеи, театры, организации → the (the British Museum, the EU).
3. Could you restart server, please?
Верно. Оба знают, какой сервер — the.
Нет. Вы оба знаете, о каком сервере речь → the.
4. unit tests are important.
Верно. Множественное число, тесты «вообще» — без артикля.
Нет. Множественное число и речь о тестах «вообще» → без артикля.
5. Caching is useful technique.
Верно. Одна из многих техник — a. Useful начинается со звука «ю» — согласного.
Нет. «Одна из многих техник» → a (useful начинается со звука «ю», это согласный звук).
6. I need hour to finish the review.
Верно. В hour буква h не читается — первый звук гласный → an.
Нет. Артикль выбирается по звуку: hour читается «ауэр» → an.
7. We need feedback from the team.
Верно. Feedback — неисчисляемое: без артикля (как information, advice).
Нет. Feedback — неисчисляемое слово (как information, advice) → без артикля.

Закрепление в реальном мире

Вернитесь в Write & Improve, откройте своё письмо про музей (или напишите заново), исправьте артикли по алгоритму — и нажмите Check. Если уровень вырос — алгоритм работает. Это займёт три минуты.

Что почитать

Главный источник урока — British Council: Articles ‘a’, ‘an’, ‘the’ (там же интерактивные упражнения — отличная добавка к этому уроку). Для глубокого погружения в the: справочник по definite article.

Если какой-то пункт алгоритма вызывает вопрос «а почему здесь так?» — спросите меня в чате. Артикли полны мелких исключений, и разбирать их лучше на ваших собственных примерах.
