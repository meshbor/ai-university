Урок 0002 · Кто ты: сессии, токены и почему 401

Курс «Senior 2026» · Фаза 0 · ~15 мин теории + практика на fullstack-lab

Сегодняшняя победа. Ты поймёшь, как сервер «помнит», что ты залогинен, хотя HTTP ничего не запоминает; в чём разница между серверной сессией и stateless JWT; что лежит внутри токена; чем 401 отличается от 403; и почему «выйти» с чистым JWT — это хитрость. Всё — на твоём реальном полигоне.

1. Теория (коротко)

Проблема: HTTP ничего не помнит

HTTP — stateless (без состояния). Каждый запрос для сервера — «чистый лист»: он сам по себе не знает, что минуту назад ты вводил пароль. Тогда как сервер понимает, что список заметок надо отдать именно тебе? Ответ: клиент при каждом запросе предъявляет «пропуск». Есть два способа сделать этот пропуск.

Два способа помнить пользователя

СпособГде живёт состояниеЧто у клиентаЦена
Сессия На сервере (память / Redis / БД) — список «кто сейчас вошёл» Короткий session_id в cookie Сервер хранит состояние; зато легко мгновенно разлогинить
JWT-токен Нигде — сервер не хранит ничего Сам токен с данными внутри (userId, role, срок) Легко масштабировать; зато трудно отозвать до истечения срока

Запомни ось различия: сессия = состояние на сервере, JWT = состояние в самом токене (stateless). Это любимый вопрос на собесе.

Что внутри JWT

JWT — это три части через точку: header.payload.signature. payload — это просто base64 с твоими данными (claims): userId, role, exp (когда протухнет). Его может прочитать кто угодно — это не шифр. А вот signature — подпись секретом сервера: подделать данные без секрета нельзя, сервер сразу заметит. Поэтому в JWT не кладут секреты — только идентификаторы.

Как это сделано в твоём lab

При POST /auth/login сервер проверяет пароль и подписывает JWT (алгоритм HS256, живёт 8 часов). Дальше токен едет к клиенту двумя путями: в HTTPOnly-cookie access_token и в теле ответа как accessToken. На каждом защищённом запросе middleware берёт токен сначала из cookie, а если её нет — из заголовка Authorization: Bearer <токен>. Нет токена или он битый → 401.

401 ≠ 403

  • 401 Unauthorized — «я не знаю, кто ты»: токен не предъявлен или невалиден. (Точнее было бы «не аутентифицирован».)
  • 403 Forbidden — «я знаю, кто ты, но тебе сюда нельзя»: прав/роли не хватает.
Тонкость безопасности из lab. Чужую заметку lab отдаёт как 404 Not Found, а не 403 — чтобы не выдавать даже сам факт её существования. Это осознанный приём, его любят обсуждать сеньоры.

Почему «logout» с JWT — хитрость

В lab logout просто стирает cookie. Но stateless-токен сервер нигде не хранит, значит и «выбросить» его не может: если ты успел сохранить Bearer-токен, он будет работать до exp. Это реальный компромисс JWT — лечат коротким сроком жизни + refresh-токенами или чёрным списком по jti. Ты увидишь это руками в практике.

4. Практика на полигоне

Теперь руками докажем, что JWT — stateless: тот же токен работает без cookie, через Bearer.

# 1. Запусти API (если ещё не запущен)
cd yamal_media/fullstack-lab/apps/api
go run ./cmd/server

# 2. Логин — и сразу достанем сам токен из тела ответа в переменную TOKEN
TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"editor@lab.local","password":"lab12345"}' \
  | grep -o '"accessToken":"[^"]*"' | cut -d'"' -f4)
echo "$TOKEN"

# 3. Загляни ВНУТРЬ токена — раскодируем payload (вторую часть). Это не взлом: payload открыт
python3 -c "import sys,base64,json;p=sys.argv[1].split('.')[1];p+='='*(-len(p)%4);print(json.dumps(json.loads(base64.urlsafe_b64decode(p)),indent=2,ensure_ascii=False))" "$TOKEN"

# 4. Кто я? Спросим БЕЗ всякой cookie — только Bearer-токеном (stateless в действии)
curl -i -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/v1/auth/me

# 5. А теперь без токена вообще — смотри статус
curl -i http://localhost:8080/api/v1/auth/me

# 6. «Выходим» (logout стирает cookie) и СНОВА бьём тем же Bearer-токеном
curl -i -X POST -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/v1/auth/logout
curl -i -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/v1/auth/me
