Урок 0003 · Контракт API: data, status, message

Курс «Senior 2026» · Фаза 0 · ~15 мин · полигон fullstack-lab (тот же формат, что yamal-media gateway)

Сегодняшняя победа. Ты научишься читать любой ответ lab (и yamal) как единый «конверт», понимать разницу между HTTP-статусом и полем status в теле, и писать на фронте один универсальный обработчик вместо угадывания «а тут JSON другой?».

1. Теория (коротко)

Зачем единый формат

Без договора каждый эндпоинт отдаёт JSON по-своему: где-то массив, где-то {error: "..."}, где-то просто строка. Фронт превращается в кашу из if. Контракт API — это договор: всегда три поля верхнего уровня. Данные — только внутри data.

Конверт (envelope) в lab

{
  "data":    { ... },   // полезная нагрузка (объект, список, пустой {})
  "status":  200,       // дублирует HTTP-статус (см. ниже зачем)
  "message": "ok"       // человекочитаемое пояснение для UI/логов
}

В коде это одна функция — все handlers зовут её, а не собирают JSON руками:

// apps/api/pkg/response/response.go
return ctx.Status(status).JSON(Envelope{
  Data: data, Status: status, Message: message,
})

Успех и ошибка — одна форма

Успех — GET /notes (фрагмент):
{"data":{"list":[...]},"status":200,"message":"ok"}
Ошибка — без cookie:
{"data":{},"status":401,"message":"unauthorized"}

Обрати внимание: при ошибке data — пустой объект {}, не null. Фронт всегда может писать res.data.list только после проверки res.ok / статуса.

HTTP-статус и body.status — зачем оба?

  • HTTP-статус (первая строка ответа: HTTP/1.1 401) — для браузера, прокси, CDN, fetch.
  • status в теле — для единообразия в шине сообщений (в yamal тот же DTO уходит через RabbitMQ).

В lab они совпадают. На собесе: «дублируем намеренно — один тип ответа и для HTTP, и для очереди».

Как фронт lab это типизирует

// apps/web/src/api/types.ts
export type ApiEnvelope<T> = {
  data: T
  status: number
  message: string
}

Один generic — и listNotes() знает, что список в data.list, а login() — что юзер в data.user.

Таблица: что класть куда (паттерн lab)

Ситуация HTTP message data
Список заметок 200 ok { list: [...] }
Создали заметку 201 created объект заметки
Нет авторизации 401 unauthorized {}
Заметка не найдена 404 note not found {}
Связь с yamal. Реальный gateway yamal-media использует тот же принцип { data, status, message } — lab специально это зеркалит. Выучив здесь, ты читаешь прод.

4. Практика на полигоне

# Логин + сохранить cookie
curl -c /tmp/lab.jar -s -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"editor@lab.local","password":"lab12345"}' | python3 -m json.tool

# Список заметок — смотри структуру data.list
curl -b /tmp/lab.jar -s http://localhost:8080/api/v1/notes | python3 -m json.tool

# Ошибка — та же форма конверта
curl -s http://localhost:8080/api/v1/notes | python3 -m json.tool

# Создать заметку — обрати внимание на HTTP 201 и message "created"
curl -b /tmp/lab.jar -s -w "\nHTTP_CODE:%{http_code}\n" -X POST http://localhost:8080/api/v1/notes \
  -H 'Content-Type: application/json' \
  -d '{"title":"Контракт API","body":"data status message"}' | python3 -m json.tool
