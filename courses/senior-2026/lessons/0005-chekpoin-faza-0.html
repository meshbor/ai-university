1. Эталонная схема (сверь со своей)

Браузер :5173 (через Vite proxy → API :8080)
① POST /api/v1/auth/login + JSON {email, password}
② API: validate → store.FindUser → CheckPassword → jwt.Sign
③ Ответ: HTTP 200 + Set-Cookie: access_token + body {data:{user, accessToken}, status:200}
④ GET /api/v1/notes + Cookie (credentials: include)
⑤ Middleware IsAuthed: cookie → Parse JWT → userId в context
⑥ Handler list → store.ListNotes(userId) → SQL
⑦ Ответ: HTTP 200 + {data:{list:[...]}, status:200, message:"ok"}

Без cookie: шаг ④ → middleware → HTTP 401 + {data:{}, status:401, message:"unauthorized"}
Неверный пароль: шаг ① → HTTP 401 + message "invalid credentials"

3. Практика: curl -v

-v (verbose) показывает заголовки запроса и ответа — учись читать их как сеньор.

# Логин — смотри > отправленные заголовки, < HTTP/1.1 200, < Set-Cookie
curl -v -c /tmp/lab.jar -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"editor@lab.local","password":"lab12345"}' 2>&1 | less

# Заметки с cookie — < HTTP/1.1 200, тело с data.list
curl -v -b /tmp/lab.jar http://localhost:8080/api/v1/notes 2>&1 | less

# Без cookie — < HTTP/1.1 401
curl -v http://localhost:8080/api/v1/notes 2>&1 | less

4. DevTools → Network (UI)

  1. Открой http://localhost:5173, F12 → Network, галочка Preserve log.
  2. Войди — найди login: Request Headers, Response Headers (set-cookie), Preview → data.user.
  3. Страница заметок — запрос notes: Request Cookie, статус 200, data.list.
  4. Выйди — снова notes → 401 (или редирект на login).
