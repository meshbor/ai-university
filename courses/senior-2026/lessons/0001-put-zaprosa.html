Урок 0001 · Путь запроса: от логина до списка заметок

Курс «Senior 2026» · Фаза 0 · ~15 мин теории + практика на fullstack-lab

Сегодняшняя победа. К концу урока ты сможешь нарисовать и объяснить полный путь запроса — от клика «Войти» до списка заметок на экране — и понимать, что означает каждый статус-код. Это фундамент всей архитектуры клиент-сервер, твоей главной оси.

1. Теория (коротко)

Запрос и ответ

Веб — это диалог. Клиент (браузер) шлёт запрос, сервер обрабатывает и шлёт ответ. Каждый запрос — это: метод + адрес + заголовки + (иногда) тело. Каждый ответ — это: статус-код + заголовки + тело.

Четыре статус-кода, которые надо знать наизусть

  • 200 — OK, всё хорошо, вот данные.
  • 401 — не авторизован: ты не доказал, кто ты (нет/невалиден токен).
  • 404 — ресурс не найден.
  • 500 — сервер сам сломался (ошибка в коде сервера).

Правило: 4xx — «виноват клиент» (неправильный запрос), 5xx — «виноват сервер».

Как сервер «помнит», что ты залогинен

При логине сервер проверяет пароль и выдаёт JWT-токен — подписанный «пропуск». В lab он кладётся в cookie (Set-Cookie: access_token=...). Дальше браузер автоматически шлёт эту cookie с каждым запросом. Нет cookie → middleware аутентификации возвращает 401.

Контракт ответа в lab (и в реальном yamal)

Любой ответ имеет единую форму: { data, status, message }. Данные всегда в поле data. Это и есть «контракт API» — договор между фронтом и бэком.

3. Собери путь запроса

Расставь шаги в правильном порядке (1–6): сценарий «логин → список заметок».

Браузер шлёт GET /notes вместе с cookie
Браузер шлёт POST /auth/login с email и паролем
Store достаёт заметки из БД → сервер отвечает 200 {data:[...]}
Сервер проверяет пароль и создаёт JWT-токен
Middleware проверяет токен из cookie (аутентификация)
Сервер кладёт токен в Set-Cookie и отвечает 200

4. Практика на полигоне

Запусти lab и пройди путь руками — это превращает теорию в навык.

# 1. Запусти API (в одном терминале)
cd yamal_media/fullstack-lab/apps/api
cp .env.example .env   # если ещё нет
go run ./cmd/server

# 2. В другом терминале — логин (сохраняем cookie в jar)
curl -c /tmp/lab.jar -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"editor@lab.local","password":"lab12345"}'

# 3. Список заметок БЕЗ cookie  → смотри статус
curl -i http://localhost:8080/api/v1/notes

# 4. Список заметок С cookie
curl -b /tmp/lab.jar http://localhost:8080/api/v1/notes
Какой статус вернул GET /notes без cookie?
