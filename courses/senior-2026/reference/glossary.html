Глоссарий

Единый словарь терминов курса. Английский термин оставляем рядом — в индустрии говорят на нём. Дополняется по мере уроков.

Клиент-сервер и HTTP

Клиент / Сервер client / server
Клиент (браузер, мобайл) шлёт запрос; сервер обрабатывает и шлёт ответ. Основа всего веба.
HTTP-метод HTTP method
GET (получить), POST (создать), PUT/PATCH (изменить), DELETE (удалить).
Статус-код status code
Число в ответе: 2xx — успех, 3xx — редирект, 4xx — ошибка клиента (401 — не авторизован, 404 — не найдено), 5xx — ошибка сервера.
Заголовки headers
Метаданные запроса/ответа: тип контента, авторизация, кэш, cookie.
Cookie / Сессия / Токен cookie / session / token
Способы «запомнить», что ты залогинен. Cookie хранится в браузере; сессия — на сервере; JWT-токен — самодостаточный подписанный пропуск.
CORS Cross-Origin Resource Sharing
Правила браузера: может ли страница с одного домена ходить в API на другом. credentials: 'include' — слать ли cookie.

API и контракт

REST REST
Стиль API: ресурсы по URL, действия — HTTP-методами. Например GET /api/v1/notes.
Контракт API API contract
Договор о форме запроса и ответа. В lab/yamal ответ единый: { data, status, message }.
Идемпотентность idempotency
Повтор одного и того же запроса не меняет результат. GET/PUT идемпотентны, наивный POST — нет.
Пагинация pagination
Отдавать данные порциями (страницами), а не всё сразу. Бывает offset- и cursor-based.
Валидация validation
Проверка входных данных до бизнес-логики (например, заголовок не короче 3 символов).

Слои бэкенда

Хендлер / Контроллер handler / controller
Принимает HTTP-запрос, достаёт параметры, зовёт бизнес-логику, формирует ответ.
Сервис / Бизнес-логика service / business logic
«Что делать» по правилам предметной области. Не знает про HTTP и про SQL напрямую.
Стор / Репозиторий store / repository
Доступ к данным (БД). Прячет SQL за методами вроде ListNotes().
Middleware middleware
Прослойка вокруг запроса: аутентификация, логирование, request_id. Срабатывает до/после хендлера.

Базы данных

Нормализация normalization
Раскладывание данных по таблицам без дублей. Денормализация — намеренные дубли ради скорости чтения.
Индекс index
Структура для быстрого поиска по столбцу. Ускоряет чтение, замедляет запись и ест место.
Транзакция transaction
Группа операций «всё или ничего» (ACID). Уровни изоляции определяют, что видят параллельные транзакции.
Проблема N+1 N+1 problem
1 запрос за списком + по запросу на каждый элемент = лавина запросов. Лечится джойном/батчингом.

Системный дизайн

Масштабирование scaling
Вертикальное — мощнее сервер; горизонтальное — больше серверов. Senior думает про горизонтальное.
Балансировщик нагрузки load balancer
Распределяет запросы между несколькими серверами.
Очередь сообщений message queue
Асинхронная передача задач между сервисами. В yamal — RabbitMQ; gateway шлёт по routing key, сервис отвечает.
Кэш cache
Быстрое хранилище частых данных (Redis, HTTP-кэш, CDN), чтобы не дёргать БД каждый раз.
Согласованность consistency
Насколько все узлы видят одинаковые данные. Бывает строгая и в конечном счёте (eventual).
Наблюдаемость observability
Логи + метрики + трейсинг. Позволяет понять, что происходит в системе под нагрузкой.
Circuit breaker circuit breaker
«Предохранитель»: если сервис падает, перестаём его дёргать, чтобы не уронить всё каскадом.
Gateway API gateway
Единая точка входа: принимает HTTP, аутентифицирует, маршрутизирует в нужный сервис. Так устроен yamal.
