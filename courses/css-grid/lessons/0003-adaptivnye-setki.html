Адаптивные сетки

Урок 0003 · ~12 минут · после урока 0002 · шпаргалка: справочник

Вы умеете собирать макеты и расставлять элементы. Сегодня — паттерн, ради которого многие вообще переходят на Grid: карточная сетка, которая сама перестраивается при изменении ширины. Одна строка CSS вместо медиазапросов на каждое количество колонок.

Знание №1: minmax()

Трек может быть не фиксированным и не долей — а диапазоном (MDN: minmax()):

grid-template-columns: minmax(200px, 1fr);

Читается: «колонка не уже 200px и не шире 1fr». Если места мало — колонка сожмётся до 200px и дальше перенос. Если места много — растянется.

Знание №2: auto-fit + repeat = магия

Свяжите repeat(), auto-fit и minmax() (CSS-Tricks: auto-fit):

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 8px;
}

Браузер сам считает, сколько колонок по 200px+ поместится, и растягивает их равномерно. Сужаете контейнер — колонок становится меньше. Расширяете — больше. Без единого @media.

Знание №3: макет страницы на узком экране

Карточки сами адаптируются. Но макет «сайдбар слева» из урока 0002 на телефоне нужно перестроить вручную — через медиазапрос и новые grid-template-areas:

@media (max-width: 600px) {
  .page {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "sidebar"
      "content"
      "footer";
  }
}

Одна колонка, четыре строки — всё друг под другом. На широком экране остаётся прежний макет из урока 0002.
